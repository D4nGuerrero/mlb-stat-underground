🌐 Foundation & Hydration

v1 (REST) REST

https://statsapi.mlb.com/api/v1/

Most endpoints live here. Teams, people, schedule, stats.

hydrate= Embed related objects. e.g., hydrate=stats,team,awards

v1.1 (GUMBO) LIVE

https://statsapi.mlb.com/api/v1.1/

Live game feed (Grand Unified Master Baseball Object). Recommended for real-time.

WebSocket WS

wss://ws.statsapi.mlb.com/api/v1.1/

Push updates via diffPatch. Connect to game feed for incremental live data.

Static Assets

https://www.mlbstatic.com/

Team logos, player headshots, and static content. Used for media delivery.

# hydrate examples (powerful single-request embedding)
GET /api/v1/teams/119?hydrate=stats,team,awards
GET /api/v1/people/660271?hydrate=stats(season,hitting,type=season)

🧠 Museum notes: All endpoints are public (no auth). Use hydrate to avoid multiple round trips. Live game polling: ~12 seconds for HTTP. WebSocket recommended for real-time. IDs reference: sportId=1 (MLB), team 119 (Dodgers), 147 (Yankees). Open-source wrappers available in Python, Rust, Ruby. Built from community research — statsapi.mlb.com.
