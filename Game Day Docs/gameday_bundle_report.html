Data flow

The bundle reads route and query-string state, merges those values into runtime config, fetches initial schedule/feed/content data, renders the app, then opens websocket subscriptions when game state and time windows allow it.

Step What happens Evidence from bundle
1 Bootstrap config resolves mount selector, env, locale, language, ads, tracking, and feature flags. mount:"#gameday-index-component__root" and query-param overrides like stitch_env, forceWS, enableFieldVision
2 Routes map into gamePk, view, tab, detail, and date/team segments. Patterns such as /:gamepk/:view/:tab/:detail
3 REST, BDFED, and GraphQL calls populate schedule, stories, matchup, metrics, and content. api/v1.1/game/.../feed/live, bdfed/gameday-content, graphql/?operationName=getGamesByGamePks
4 Websocket subscriptions stream live push updates and patch payloads. api/v1/game/push/subscribe/gameday/{gid}, new WebSocket(e, ["gameday", token])
5 Components resolve visual media through static URL registries and runtime player/media links. teamLogo, mugshot, playerHeader, videoPlayer, mlbtvUrl

How to recreate the pitch trail effect

The trailing pitch-path look is just layered drawing with temporal memory. You sample a sequence of points from a parametric trajectory, keep earlier points around for a short time, and draw older segments thinner and more transparent than newer segments.

Core drawing recipe

  • Model the pitch as a time-ordered list of points from release to plate.
  • On each animation frame, reveal more of the list.
  • Draw line segments between visible points.
  • Fade alpha and line width based on segment age.
  • Add a brighter leading ball marker at the newest point.
  • Optionally overlay a strike zone and plate for spatial context.

What makes it feel like PitchFX

  • Curvature from horizontal and vertical break, not straight-line interpolation.
  • Uneven timing so the pitch appears to accelerate into the zone.
  • Head glow plus fading tail, not a single opaque stroke.
  • Depth cueing from thicker recent segments and dimmer older ones.
  • Stable coordinate transform so every pitch lands inside the same zone frame.
Minimal algorithm
1. Build N points for the pitch path.
2. Each frame, increase visiblePointCount.
3. For each visible segment:
   - age = segmentIndex / visiblePointCount
   - alpha = lerp(0.12, 0.95, age)
   - width = lerp(2, 7, age)
   - stroke segment with that alpha and width
4. Draw the newest point as a glowing baseball.
5. Clear and redraw every frame.
Canvas snippet
for (let i = 1; i < visibleCount; i++) {
  const age = i / (visibleCount - 1);
  ctx.strokeStyle = `rgba(255, 187, 71, ${0.12 + age * 0.83})`;
  ctx.lineWidth = 2 + age * 5;
  ctx.beginPath();
  ctx.moveTo(points[i - 1].x, points[i - 1].y);
  ctx.lineTo(points[i].x, points[i].y);
  ctx.stroke();
}

const head = points[visibleCount - 1];
ctx.fillStyle = "#fff6d6";
ctx.shadowColor = "rgba(255, 196, 84, 0.95)";
ctx.shadowBlur = 18;
ctx.beginPath();
ctx.arc(head.x, head.y, 6, 0, Math.PI * 2);
ctx.fill();
